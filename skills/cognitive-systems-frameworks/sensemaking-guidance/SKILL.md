# 意义建构引导 · Sensemaking Guidance

## 一句话功能

在模糊、不确定或危机情境中，引导人们从"不知道发生了什么"走向"可行动的理解"——通过行动、回顾和叙事来建构意义。

## 何时使用

- 团队面临模糊情境，不确定"发生了什么"
- 组织经历重大变化或危机
- 信息不足以支撑分析性决策
- 人们对同一事件有截然不同的理解

## 何时不使用

- 信息充分、情况明确——使用理性分析即可
- 需要快速执行而非理解
- 问题是技术性的（有明确答案）

## 理论基础

- 来源思想家：[维克 Weick](../../domains/cognitive-systems/schools/naturalistic-decision/weick.md)
- 来源概念：[意义建构](../../domains/cognitive-systems/concepts/sensemaking.md)
- 关键文献：Weick, K. *Sensemaking in Organizations*. Sage, 1995.

## 操作流程

### Step 1：诊断意义建构状态
评估当前的意义建构状态：
- **有没有意义？** 人们是否知道"发生了什么"？
- **意义是否在崩溃？** 原有理解是否正在失效？
- **有没有行动？** 人们是在行动（建构理解）还是冻结？
- **身份是否受到威胁？** "我是谁"是否正在被挑战？

### Step 2：促进行动（Enactment）
如果人们冻结了——需要促进"通过行动来理解"：
- **小步行动**：不要等待完美方案，先做小步尝试
- **标记行动**：为行动命名——"我们现在在做的是 X"
- **制定环境**：行动本身会改变环境，使新的信息浮现
- **提问范式**：
  - "如果我们不做什么，最坏会怎样？"
  - "最小的下一步是什么？"
  - "做了之后我们可能学到什么？"

### Step 3：促进回顾（Retrospection）
如果人们在行动但无法理解——需要促进"回顾性理解"：
- **叙事**：把发生了什么讲成一个故事
- **模式识别**：在过去是否见过类似的模式？
- **线索提取**：环境中有哪些小线索可以拼成大图景？
- **提问范式**：
  - "回头看，什么让你感到意外？"
  - "如果这是一部电影，现在是什么情节？"
  - "我们之前什么时候见过类似的信号？"

### Step 4：促进社会互动
意义建构是社会过程——需要促进对话：
- **多元视角**：邀请不同背景的人参与
- **安全空间**：允许"我不知道"和"我有不同的理解"
- **共同叙事**：从多个故事中找到共同线索
- **提问范式**：
  - "你看到了什么我没有看到的？"
  - "如果你的理解是对的，那意味着什么？"
  - "我们能否找到一个大家都觉得'说得通'的故事？"

### Step 5：形成可行动的理解
目标不是"完美理解"，而是"可行动的可信叙事"：
- **可信度检查**：这个理解"说得通"吗？
- **可行动性检查**：基于这个理解，我们能做什么？
- **持续修正**：行动后再回顾——理解需要持续更新

## 完整示例

**输入场景**：
> 一家 SaaS 公司的核心产品突然出现大量用户流失，原因不明。团队陷入恐慌，不知道"发生了什么"。

**Skill 应用过程**：

1. **诊断**：意义正在崩溃——原有的"产品很好"的理解失效了；团队冻结——每个人有不同猜测但无人行动

2. **促进小步行动**：
   - 让客服团队主动联系 20 个流失用户（小步行动）
   - 标记："这是我们的'诊断行动'，不是'挽救行动'"

3. **促进回顾**：
   - 回顾近 3 个月的产品变更——发现一次 API 升级
   - 回顾用户反馈——发现多条抱怨"集成断裂"的工单
   - 线索拼合：API 升级→第三方集成断裂→用户流失

4. **社会互动**：
   - 工程团队看到了技术细节，销售团队看到了客户情绪
   - 共同叙事："我们在追求新功能时忽视了稳定性，现在需要修复信任"

5. **可行动理解**：
   - 可信度：说得通——API 变更时间与流失曲线匹配
   - 可行动性：回滚 API 变更、主动联系受影响用户、发布修复说明

**输出**：
> 通过小步行动→回顾→社会互动，团队从"不知道发生了什么"走向了"API 升级导致集成断裂"的可行动理解。

## 反例（误用）

**误用场景**：
> 在信息充分的危机中（如明确的火灾警报），花时间做意义建构——此时需要的是执行而非理解。

**正确做法**：区分"需要理解"和"需要执行"——前者用意义建构，后者用标准操作规程。
