# 活动系统分析 · Activity System Analysis

## 一句话功能

用恩格斯托姆的三角形模型系统分析一个活动系统（工作团队、教育场景、组织流程），找到结构性矛盾并设计变革干预。

## 何时使用

- 分析组织/团队工作中的系统性问题
- 理解技术引入后的系统性影响
- 设计组织变革干预
- 分析教育/学习活动的结构

## 何时不使用

- 问题是个体层面的（心理/能力问题）
- 需要快速修复而非系统分析
- 活动边界模糊难以确定

## 理论基础

- 来源思想家：[恩格斯托姆 Engeström](../../学派/分布式认知/engestrom.md)
- 来源概念：[活动理论](../../概念/活动理论.md)
- 关键文献：Engeström, Y. *Learning by Expanding*. Cambridge University Press, 1987/2015.

## 操作流程

### Step 1：确定活动系统边界
明确要分析的活动系统：
- **活动是什么？** 这个系统存在的目的是什么？（客体/目标）
- **谁参与？** 主体是谁？共同体包括谁？
- **边界在哪里？** 这个活动与相邻活动的区分

### Step 2：绘制三角形模型
填入六个要素：

```
            工具 (Tools)
           /              \
          /                \
    主体 (Subject) ——→ 客体 (Object) → 结果 (Outcome)
          \                /
           \              /
    规则 (Rules) — 共同体 (Community) — 分工 (Division of Labor)
```

- **主体**：谁在从事这个活动？
- **客体**：活动的目标/对象是什么？
- **工具**：使用什么工具（物理的、符号的、数字的）？
- **规则**：有什么规则、规范、惯例？
- **共同体**：活动的社会语境是谁？
- **分工**：任务如何分配？权力如何分配？

### Step 3：识别矛盾
检查三角形内部和之间的矛盾：
- **一级矛盾**（要素内部）：同一要素内部的冲突（如：工具既支持又阻碍工作）
- **二级矛盾**（要素之间）：不同要素之间的冲突（如：新工具与旧规则不匹配）
- **三级矛盾**（活动之间）：当前活动与新引入活动之间的冲突
- **四级矛盾**（系统之间）：不同活动系统之间的冲突

### Step 4：矛盾→变革
将矛盾视为变革的机会：
- 矛盾是否可以被利用来推动扩展学习？
- 什么是可能的"扩展"方向？（重新定义客体？引入新工具？改变规则？）
- 如何让活动系统的参与者共同分析和改造系统？

## 完整示例

**输入场景**：
> 一所大学引入在线学习平台后，教师和学生的满意度都下降了。分析这个活动系统的问题。

**Skill 应用过程**：

1. **活动系统边界**：大学教学活动——客体是"培养有能力的毕业生"

2. **三角形绘制**：
   - 主体：教师
   - 客体：培养学生能力（从"传授知识"向"培养能力"转化中）
   - 工具：传统教材 + 新引入的在线平台
   - 规则：学分制、考试评估、课时要求
   - 共同体：院系、学生、管理层
   - 分工：教师讲、学生听、管理评估

3. **矛盾识别**：
   - 二级矛盾：新工具（在线平台支持互动学习）vs 旧规则（固定课时+纸笔考试）→ 教师想互动但评估还是考试
   - 二级矛盾：新客体（培养能力）vs 旧分工（教师讲学生听）→ 能力需要实践，但分工仍是单向传递
   - 三级矛盾：教学活动 vs 在线平台设计者预设的活动模式

4. **变革方向**：
   - 利用矛盾推动扩展：重新定义评估方式（从考试到项目）
   - 改变分工：从"教师讲"到"师生共同建构"
   - 让教师参与平台定制

**输出**：
> 满意度下降不是工具问题，而是活动系统的结构性矛盾——新工具与旧规则、旧分工的不匹配。解决方案不是更好的工具培训，而是活动系统的整体重组。

## 反例（误用）

**误用场景**：
> 用活动理论分析个人的心理困惑——活动理论是系统分析工具，不是心理治疗工具。

**正确做法**：个人心理问题使用心理学框架（如 CBT），组织/活动问题使用活动理论。
