# 拉斯穆森 / Jens Rasmussen

> **领域**: 认知工程 / 系统安全
> **时期**: 现代 (Modern)
> **生卒**: 1926–2018
> **国籍**: 丹麦

> "人在复杂系统中的行为，不能用简单的刺激-反应来理解。"

## 一、思想概要

延斯·拉斯穆森（Jens Rasmussen, 1926–2018）是丹麦Risø国家实验室认知系统工程研究组创始人，被公认为认知系统工程（CSE）的创始人之一。他的核心贡献在于为理解复杂系统中人的认知行为提供了系统性的分析框架——特别是SRK（技能-规则-知识）三层次模型和抽象层级（Abstraction Hierarchy）工具。

拉斯穆森在核电厂安全研究中认识到，传统的可靠性工程（概率分析）无法解释操作员在紧急情况下的认知行为。这一认识驱使他发展出一套全新的认知工作分析方法（CWA），将人因工程从"刺激-反应"的简化模型提升到系统性的认知分析。

他的思想直接影响了Vicente的生态界面设计、Hollnagel的CREAM方法、Woods的韧性工程，以及Leveson的系统安全理论。在认知系统工程领域，拉斯穆森的理论框架至今仍是基础性的参考架构。

Jens Rasmussen founded cognitive systems engineering at Risø National Laboratory in Denmark. His SRK model (Skill-Rule-Knowledge) and Abstraction Hierarchy remain foundational tools for analyzing cognitive work in complex systems. His work transformed human factors from stimulus-response models to systematic cognitive analysis.

## 二、核心命题

1. **SRK模型**：人的行为有三个层次——技能型（Skill-based）、规则型（Rule-based）、知识型（Knowledge-based）。不同层次的行为需要不同的界面设计和错误分析。
2. **抽象层级（Abstraction Hierarchy）**：复杂系统可以用五个抽象层级来描述——从物理形式到功能目的。认知工作分析的核心工具。
3. **错误不是随机缺陷**：人的错误是系统性认知策略在特定条件下的可预测结果——理解错误需要理解认知策略。
4. **生态界面设计**：界面应呈现系统的深层结构，而非仅仅表面数据——让操作者"直接感知"系统状态。
5. **事故因果的层级模型（AcciMap）**：事故不是单一原因造成的，而是多层防御的联合失效。

## 三、思想脉络

### 早期（核安全与可靠性）
在Risø国家实验室研究核电厂安全——发现传统可靠性工程无法解释操作员在紧急情况下的认知行为。

### 中期（SRK模型与认知工作分析）
1983年发表里程碑论文"Skills, Rules, and Knowledge"，提出行为三层次模型。发展认知工作分析（CWA）方法论。

### 晚期（事故模型与韧性）
提出AcciMap事故因果层级模型，关注系统层面的安全——不只是个人错误，而是多层防御的联合失效。

## 五、重要概念

- **SRK模型**：技能型（自动化、无需注意）→规则型（遵循已知规则）→知识型（需要分析和推理）
- **抽象层级**：功能目的→价值/优先性→一般化功能→物理功能→物理形式
- **认知工作分析 CWA**：分析工作领域的认知需求——五步框架
- **生态界面设计 EID**：界面呈现应匹配系统的深层结构
- **AcciMap**：事故因果的层级分析模型

## 六、思想坐标

- **承接**：维纳（控制论）、吉布森（生态心理学）
- **批判**：传统人因工程（仅关注操作层）、概率安全分析（忽略认知）
- **影响**：Woods（韧性工程）、Vicente（生态界面设计）、Hollnagel（CREAM）、Leveson（STAMP）

## 七、当代应用

- **核安全与航空安全**：CWA是核电厂和航空领域工作系统设计的标准方法
- **医疗安全**：分析医疗工作中的认知需求和错误模式
- **AI界面设计**：AI系统的界面应呈现其推理过程（抽象层级）
- **自动驾驶**：理解驾驶员在不同自动化级别下的认知需求

## 八、常见误读

- **"SRK模型是静态分类"**：三个层次是动态切换的——当规则失效时，人从规则型切换到知识型。
- **"CWA只是任务分析"**：CWA分析的是工作领域的认知约束和可能性，不仅仅是具体任务。
- **"错误分类是为了惩罚"**：错误分类是为了理解系统性的认知约束，从而改进设计。
