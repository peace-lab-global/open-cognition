# 诺曼 Donald Norman

> "不是人犯了错，而是设计引诱了错误。"

## 基本信息

- **生卒年**：1935–
- **国籍/文化背景**：美国
- **学术身份**：加州大学圣迭戈分校名誉教授，尼尔森·诺曼集团联合创始人
- **学派归属**：认知工效学、以人为中心的设计

## 核心命题

1. **不是人犯错，是设计犯错**：大多数"人为错误"的根因是糟糕的设计——好的设计让正确行为自然发生，让错误行为难以发生。
2. **行动七阶段模型**：人的行动可以分解为七个阶段——从目标形成到评估结果。每个阶段都可能出现错误，好的设计在每个阶段提供支持。
3. **可供性（设计中的 Affordance）**：好的设计让功能"可见"——按钮"看起来"可以被按，把手"看起来"可以被拉。
4. **情感化设计**：设计不仅关乎功能，还关乎情感——美观的界面让人感觉更好，也更易用（积极情感扩展认知灵活性）。

## 思想脉络

### 早期（认知心理学）
在哈佛大学和加州大学圣迭戈分校研究记忆和注意——提出"记忆中的信息"如何被检索和组织。

### 中期（设计心理学与认知工效学）
《设计心理学》（1988，原名 *The Psychology of Everyday Things*）将认知心理学引入设计领域。《人为错误》（1990）系统分析错误的类型和设计对策。

### 晚期（情感化设计与以人为本）
《情感化设计》（2004）将情感纳入设计框架。推动"以人为中心的设计"成为全球设计实践的主流。

## 重要概念

- **行动七阶段**：目标 → 意图 → 行动规划 → 执行 → 感知状态 → 解释 → 评估
- **可供性 Affordance**（设计语境）：设计元素应让功能"可见"——与 Gibson 的原义有差异
- **约束 Constraints**：通过物理、逻辑、文化约束减少错误可能
- **映射 Mapping**：控制与效果之间应有清晰的对应关系
- **错误宽容 Error Tolerance**：设计应宽容错误——可撤销、可恢复

## 思想坐标

- **承接**：吉布森（可供性）、西蒙（有限理性）、认知心理学
- **批判**：以技术为中心的设计、"用户太笨"的假设
- **影响**：UX 设计、人机交互、产品设计、服务设计

## 当代应用

- **UX/UI 设计**：诺曼的设计原则是全球 UX 设计师的基本功
- **AI 界面**：AI 系统如何让用户理解其能力边界——避免过度信任或不信任
- **医疗设计**：医疗设备界面的设计错误可能导致致命后果
- **自动驾驶**：如何让驾驶员理解自动驾驶的状态和能力边界

## 常见误读

- ❌ "Norman 的 affordance = Gibson 的 affordance" → Norman 把 affordance 理解为"感知到的功能提示"（perceived affordance），Gibson 的原义是"环境提供的行动可能性"——两者有重要区别。
- ❌ "以人为中心 = 用户说了算" → 以人为中心是理解用户需求，但设计需要平衡用户、技术和商业。
- ❌ "设计心理学 = 美化界面" → 设计心理学关注的是认知过程——如何让界面匹配人的认知方式。
