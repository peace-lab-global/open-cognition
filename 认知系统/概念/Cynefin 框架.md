# Cynefin 框架

## 一句话定义

Cynefin 框架是一种情境感知工具，将问题空间按"因果关系可理解性"与"模式可识别性"分为五个域（简单/繁杂/复杂/混沌/失序），帮助决策者判断"此刻该用哪种思维方法"——它是选择思维方法的元框架。

## 提出者与背景

- **首要提出者**：Dave Snowden，威尔士知识管理学者，名称取自威尔士语"cynefin"（栖息地/归属感）
- **合作者**：Mary Boone 将其引入管理实践；Cynthia Kurtz 在叙事研究中共同发展
- **历史语境**：传统管理理论假设所有问题都可用同一种"最佳实践"解决，忽视了不同复杂性层级的根本差异
- **针对的问题**：如何避免"用简单域的方法解决复杂域的问题"——这是组织失败最常见的原因之一

## 核心要义

### 五域模型

```
                    因果关系
              可感知              不可感知
         ┌─────────────┬──────────────────┐
    可    │             │                  │
    识    │  简单域     │   混沌域         │
    别    │  (Simple)   │  (Chaotic)       │
    │     │             │                  │
    模    ├─────────────┼──────────────────┤
    式    │             │                  │
         │  繁杂域     │   复杂域         │
         │ (Complicated)│  (Complex)       │
         │             │                  │
         ├─────────────┼──────────────────┤
         │        失序域 (Disorder)        │
         │       ← 你不知道自己在哪个域 →   │
         └─────────────┴──────────────────┘
```

### 五域决策策略

| 域 | 因果关系 | 模式 | 决策策略 | 典型方法 |
|----|----------|------|----------|----------|
| **简单** (Clear) | 可预测 | 稳定 | **感知→分类→响应** | 最佳实践、检查清单、SOP |
| **繁杂** (Complicated) | 可分析 | 可识别 | **感知→分析→响应** | 专家诊断、[麦肯锡方法](./mckinsey-method.md) |
| **复杂** (Complex) | 不可预测 | 涌现 | **探测→感知→响应** | 安全失败实验、[设计思维](./design-thinking.md) |
| **混沌** (Chaotic) | 不可知 | 无 | **行动→感知→响应** | 果断行动、[OODA](./ooda-loop.md)、强领导力 |
| **失序** (Disorder/Confused) | 不确定 | 不确定 | 先判断自己在哪个域 | 元认知、[意义建构](./sensemaking.md) |

### 关键论点

1. **最佳实践只在简单域有效**：将简单域的"最佳实践"移植到复杂域是灾难性的
2. **繁杂 ≠ 复杂**：繁杂（如造飞机）可以被分解分析；复杂（如养育孩子）只能被探测适应
3. **失序域是最危险的**：当你不知道自己不知道什么时，首先需要判断自己在哪个域
4. **域的动态迁移**：问题可能随时间在域间移动（如技术成熟后，复杂域问题变为繁杂域）
5. **安全失败（Safe-to-Fail）**：在复杂域中，不追求"不犯错"，而追求"错误成本可控"

## 通俗用法 vs 学术原义

| 维度 | 通俗用法 | 学术原义 |
|------|----------|----------|
| "判断复杂度" | "看看问题有多难" | 诊断问题空间的因果结构与模式可识别性 |
| "用对方法" | "因地制宜" | 不同因果结构需要不同的认识论策略 |
| "最佳实践" | "照着成功案例做" | 仅在简单域（因果关系稳定且可知）中有效 |

## 与相关概念的关系

- **元框架**：Cynefin 是"选择思维方法的框架"——它告诉你此刻该用哪个工具
- **互补**：[系统思维](./systems-thinking.md) — 系统思维处理复杂域问题，Cynefin 帮你识别"这是不是复杂域问题"
- **互补**：[麦肯锡方法](./mckinsey-method.md) — 麦肯锡适合繁杂域，Cynefin 警告你不要把它误用在复杂域
- **互补**：[OODA 循环](./ooda-loop.md) — OODA 适合混沌域的快速响应
- **关联**：[意义建构](./sensemaking.md) — 失序域中 Cynefin 引导的正是意义建构过程
- **关联**：[有限理性](./bounded-rationality.md) — Cynefin 帮助识别有限理性的边界在哪里

## 代表思想家

- **Dave Snowden** — Cynefin 创始人，将复杂性科学引入组织知识管理
- **Mary Boone** — 将 Cynefin 引入哈佛商业评论，使其进入主流管理话语
- **Cynthia Kurtz** — 叙事研究与 Cynefin 的结合
- **Kurtz & Snowden 的 IBM 研究** — 在 IBM 全球服务部门验证 Cynefin 的实践效果

## 应用场景

- **组织变革**：诊断变革问题的复杂性层级——简单问题用流程，复杂问题用实验
- **产品开发**：判断产品问题是繁杂（技术问题，可分析）还是复杂（市场问题，需探测）
- **危机管理**：混沌域中需要果断行动而非完美分析
- **知识管理**：区分"显性知识"（简单/繁杂域）和"隐性知识"（复杂域）
- **政策制定**：避免用"最佳实践"思维处理复杂社会问题

## 常见误读

- **误读 1**："Cynefin 只是一个 2×2 矩阵" — 它的价值在于动态迁移和失序域的元认知，而非静态分类
- **误读 2**："复杂域不能用分析方法" — 分析方法仍然有用，但不能作为唯一手段；需要与实验性探测结合
- **误读 3**："简单域不需要思考" — 简单域的问题曾经也是复杂的，是经验积累使其"简单化"
- **误读 4**："Cynefin 可以替代其他思维工具" — 它是元框架，用于选择工具而非替代工具
