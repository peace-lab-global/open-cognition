# 质量标准 · Quality Criteria

本标准用于 PR 审核与维护者自检。所有条目需通过以下检查方可合并。

## 一、事实性

- [ ] 出生年、卒年、著作出版年与权威来源（Stanford Encyclopedia of Philosophy / Encyclopedia Britannica / 学术专著）一致
- [ ] 概念定义与原典或权威研究一致；通俗化表达必须明确标注"通俗用法"
- [ ] 不杜撰未发生过的逸事或未说过的话
- [ ] 引用原文必须给出出处（书名/章节/边码）

## 二、完整性

- [ ] frontmatter 字段完整（id / title / type / domain / school / era / tags / aliases / sources）
- [ ] 至少 2 条权威引用
- [ ] 至少 1 条跨领域互链（除非主题确属本领域独有）
- [ ] 思想家条目必须包含其核心概念列表；概念条目必须包含其代表思想家列表

## 三、辨析性

- [ ] 区分"通俗用法 vs 学术原义"
- [ ] 列出至少 1 个常见误读
- [ ] 争议性论点呈现至少两方立场，注明各自代表学者
- [ ] 中英文术语对照（首次出现时给出原文）

## 四、互链规范

- [ ] 跨条目链接使用相对路径
- [ ] 跨学科链接旁标注关联类型（见 [TAGS.md](../TAGS.md#关联类型-relation-types)）
- [ ] 同一关联在双向条目中均出现（如荣格条目链接弗洛伊德，弗洛伊德条目也需链接荣格）

## 五、Skill 专项

- [ ] YAML frontmatter 含 `name` `description`，description 中明确何时触发
- [ ] 含"何时使用 / 何时不使用"清单
- [ ] 至少 1 个完整使用示例
- [ ] 至少 1 个反例（误用场景及说明）
- [ ] 链接到 `domains/` 下对应的理论条目

## 六、风格

- [ ] 拒绝主观评价词（"伟大""可怕""愚蠢"），改用描述性表达
- [ ] 拒绝传教式语言；宗教条目以学术中立视角呈现
- [ ] 拒绝原创理论或个人感悟
- [ ] 段落简洁，避免冗长复述

## 七、自检命令清单（手动）

1. 在仓库根目录确认条目已加入 [INDEX.md](../INDEX.md)
2. 检查所有标签是否登记于 [TAGS.md](../TAGS.md)
3. 跟随每个内部链接验证目标文件存在
4. 检查是否在双向条目中都补全了关联

未通过本清单的 PR，维护者将打回修改。
